## Sektion 4: Monitor-Integration — Datenfluss-Diagramm

```
Editor setzt target → store.setLayoutTarget(layoutId, {view, placement})
  │
  ├─ Target-Unikat-Enforcer:
  │   Sucht Layout mit gleichem view+placement
  │   → Falls vorhanden: DESSEN target wird auf undefined gesetzt
  │   → Sync beider Layouts zum Server
  │
  ├─ inlineMonitorPanelsCrossZone (computed)
  │   Filter: view='monitor', placement='inline', scope!='zone'
  │   Sortiert nach target.order
  │   → MonitorView L1: InlineDashboardPanel nach Zone-Tiles + ActiveAutomations
  │
  ├─ inlineMonitorPanelsForZone(zoneId) (FUNKTION, nicht computed)
  │   Filter: view='monitor', placement='inline', scope='zone', zoneId=match
  │   → MonitorView L2: inlineMonitorPanelsL2 = cross-zone + zone-spezifische (dedupliziert)
  │   → Rendert nach Sensoren/Aktoren/Regeln/Zone-Dashboards
  │
  ├─ sideMonitorPanels (computed)
  │   Filter: view='monitor', placement='side-panel'
  │   → MonitorView: <aside class="monitor-layout__side"> sticky rechts
  │   → Grid: 1fr 300px (Desktop), stacked (Mobile)
  │
  ├─ bottomMonitorPanels (computed) [EXISTIERT: JA]
  │   Filter: view='monitor', placement='bottom-panel'
  │   → MonitorView: <div class="monitor-layout__bottom">
  │   → max-height: 400px, overflow-y: auto
  │   → mode="inline" (CSS-Grid, nicht GridStack)
  │
  └─ hardwarePanels (computed)
      Filter: view='hardware'
      → HardwareView: Side-Panel sticky rechts
```

### L1 Reihenfolge (MonitorView)

```
1. Zone-Tiles Grid (40px margin-bottom)
2. Aktive Automatisierungen (ActiveAutomationsSection)
3. Dashboard Overview Card (Cross-Zone Dashboard Chips)
```

## Sektion 8: Offene Fragen

### 8.1 — CHART_COLORS Palette

Die exakten Farbwerte der `CHART_COLORS` Palette (in `WidgetConfigPanel.vue` fuer Farbauswahl und in `MultiSensorWidget.vue` fuer Sensor-Farben) konnten nicht vollstaendig aufgeloest werden. Sie werden aus einer separaten Datei (`utils/chartColors.ts` oder aehnlich) importiert. **Luecke:** Datei wurde nicht explizit gelesen.

### 8.2 — AlarmListWidget Zeilenzahl

Die exakte Zeilenzahl der `AlarmListWidget.vue` wurde auf ~320 geschaetzt, aber nicht exakt verifiziert. Die Datei wurde inhaltlich analysiert, die Zeilenanzahl aber nicht einzeln gezaehlt.

### 8.3 — Sensor-Daten-Aggregation: Strategische Entscheidung

Es fehlt eine strategische Entscheidung:
- **Option A:** resolution-Parameter auf `GET /sensors/data` (Backend-Aggregation, min/max/avg pro Zeitfenster)
- **Option B:** Frontend-Downsampling (Rohwerte laden, im Browser aggregieren)
- **Option C:** Separater Aggregations-Endpoint (CQRS-Pattern)

Der HistoricalChartWidget laedt aktuell Rohwerte mit `limit=100` Default. Bei langen Zeitraeumen (7d) koennte das unzureichend sein.

### 8.4 — Dashboard-Sharing-UI

Das Backend unterstuetzt `is_shared: boolean` auf Dashboards, aber es gibt **kein UI** im Frontend, um Dashboards mit anderen Usern zu teilen. Die Funktionalitaet existiert nur im Backend.

### 8.5 — `scope` ohne `zoneId`

Wenn ein User manuell ein Dashboard mit `target.placement='inline'` erstellt, wird `scope` nicht automatisch gesetzt. Die Frage: Soll `scope` aus `target` abgeleitet werden, oder muss der User beides manuell setzen?

### 8.6 — monitorRouteForLayout und scope-Requirement

`monitorRouteForLayout` (computed in CustomDashboardView) gibt `null` zurueck wenn `layout.scope` nicht gesetzt ist. Das bedeutet: Der "Im Monitor anzeigen"-Button ist **unsichtbar** fuer Dashboards ohne scope — auch wenn ein target gesetzt ist. **Moeglicher Bug:** Target gesetzt, aber Monitor-Link nicht sichtbar.

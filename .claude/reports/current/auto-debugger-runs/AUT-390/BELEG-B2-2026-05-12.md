# BELEG — B2: columns='' → 200 statt 422 im Sensor-Export-Endpoint

## Symptom-Zusammenfassung

`GET /api/v1/sensors/export?esp_id=X&columns=` gibt HTTP 200 mit der Standard-Spalten-Header-Zeile zurück, anstatt 422 zu liefern. Ein explizit leerer `columns=`-Parameter sollte als ungültige Eingabe behandelt werden (keine Spalten ausgewählt = kein sinnvoller Export), wird aber fälschlicherweise als "kein Parameter übergeben" interpretiert und fällt auf den Default zurück.

## Code-Beleg

`El Servador/god_kaiser_server/src/api/v1/sensors.py:1689`

```python
# Validate and parse column selection
if columns:                                          # ← BUG: "" ist falsy
    col_list = [c.strip() for c in columns.split(",") if c.strip()]
    unknown = sorted(set(col_list) - _EXPORT_ALLOWED_COLUMNS)
    if unknown:
        raise HTTPException(
            status_code=422,
            detail=f"Unbekannte Spalten: {unknown}. Erlaubt: {sorted(_EXPORT_ALLOWED_COLUMNS)}",
        )
else:
    col_list = list(_EXPORT_DEFAULT_COLUMNS)         # ← columns="" landet hier
```

## Kausalerklärung

FastAPI liefert `columns=""` als leeren String (nicht `None`). Der `if columns:` Check ist ein Python-Falsy-Check — ein leerer String ist falsy, also wird der `else`-Zweig ausgeführt und die Default-Spalten verwendet. Korrekt wäre `if columns is not None:`, dann würde `col_list` für `columns=""` eine leere Liste ergeben und der nachgelagerte Code sollte mit `if not col_list: raise HTTPException(422, ...)` abgefangen werden.

## Fix-Vorschlag (Minimal)

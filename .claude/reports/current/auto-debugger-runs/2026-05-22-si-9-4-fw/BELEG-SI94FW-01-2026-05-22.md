## Befund

OfflineModeManager implementiert eine 5-State-Machine (nicht 4, wie in AUT-426 genannt — ADOPTING-State ist ein fuenfter State). Die vollstaendige Transitions-Tabelle ist:

| Trigger | Guard | Von | Nach | Code-Zeile (.cpp) |
|---------|-------|-----|------|-------------------|
| `onDisconnect()` aufgerufen | mode_ == ONLINE | ONLINE | DISCONNECTED | 247-252 |
| `onDisconnect()` aufgerufen | mode_ == RECONNECTING | RECONNECTING | OFFLINE_ACTIVE | 253-268 |
| `onDisconnect()` aufgerufen | mode_ == ADOPTING | ADOPTING | DISCONNECTED (grace neu starten) | 269-284 |
| `checkDelayTimer()`: millis()-disconnect_timestamp_ms_ >= 30000 | mode_ == DISCONNECTED | DISCONNECTED | OFFLINE_ACTIVE | 482-484 (via activateOfflineMode()) |
| `onReconnect()` aufgerufen | mode_ == OFFLINE_ACTIVE | OFFLINE_ACTIVE | RECONNECTING | 288-293 |
| `onReconnect()` aufgerufen | mode_ == DISCONNECTED (grace) | DISCONNECTED | ONLINE | 294-299 |
| `onServerAckReceived()`: Epoch valid, nicht-Duplikat | mode_ == RECONNECTING oder OFFLINE_ACTIVE | RECONNECTING/OFFLINE_ACTIVE | ADOPTING | 352-402 (via enterAdoptingMode()) |
| `onServerAckReceived()` | mode_ == DISCONNECTED | DISCONNECTED | ONLINE (grace-cancel) | 416-422 |
| `checkDelayTimer()`: adoption_started_ms_ > 0 && elapsed >= 2000 | mode_ == ADOPTING | ADOPTING | ONLINE | 466-470 (via finalizeAdoptingMode() -> deactivateOfflineMode()) |
| `onEmergencyStop()` aufgerufen | beliebig | ANY | ONLINE | 444-458 |

Zusaetzlich: OFFLINE_ACTIVE -> OFFLINE_ACTIVE moeglich wenn Reconnect-Versuch waehrend RECONNECTING erneut disconnect ausloest (RECONNECTING -> OFFLINE_ACTIVE via onDisconnect).

## Code-Evidenz

```cpp
// offline_mode_manager.h Zeile 16-22: Enum-Definition
enum class OfflineMode : uint8_t {
    ONLINE         = 0,
    DISCONNECTED   = 1,
    OFFLINE_ACTIVE = 2,
    RECONNECTING   = 3,
    ADOPTING       = 4   // <-- 5. State, in AUT-412/AUT-426 als "4-State" beschrieben
};

// offline_mode_manager.h Zeile 86-88:
bool isOfflineActive() const {
    return mode_ == OfflineMode::OFFLINE_ACTIVE || mode_ == OfflineMode::RECONNECTING;
}
```

## Doku-Drift

AUT-412 Body und der Header-Kommentar in offline_mode_manager.h (Zeile 13-14) beschreiben den Manager als "4-state state machine". Tatsaechlich existieren 5 States (ONLINE, DISCONNECTED, OFFLINE_ACTIVE, RECONNECTING, ADOPTING). ADOPTING war eine spaetere Erweiterung.

Der Header-Kommentar lautet:
```
// Implements a 4-state state machine for connectivity-aware
// offline hysteresis control.
```

## Kanonisch-Bewertung

J — Die Implementierung ist korrekt und vollstaendig. Nur die Beschreibung (Kommentar im Header + AUT-426/AUT-412 Mentions) ist falsch (4 statt 5 States). Kein Code-Fix erforderlich, nur Doku-Update.

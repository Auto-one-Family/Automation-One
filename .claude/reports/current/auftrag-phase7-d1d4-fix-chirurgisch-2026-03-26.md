## Fix-F4: Zone-Mini-Widgets per Tile-Dashboard befuellen

### Problem

Die ZoneTileCard hat einen `extra`-Slot in dem Mini-Widgets (Gauge, Sensor-Card) als kompakte InlineDashboardPanels gerendert werden sollen. Die gesamte Rendering-Infrastruktur ist vorhanden und funktioniert:

| Baustein | Datei | Zeile | Status |
|----------|-------|-------|--------|
| `#extra`-Slot in ZoneTileCard | `ZoneTileCard.vue` | 84-85 | Existiert |
| `v-if="getZoneMiniPanelId(zone.zoneId)"` | `MonitorView.vue` | 1652 | Existiert, gibt aber `undefined` zurueck |
| `<InlineDashboardPanel :panelId="..." compact>` | `MonitorView.vue` | 1653-1657 | Existiert im Template |
| `getZoneMiniPanelId(zoneId)` | `MonitorView.vue` | 940-948 | Logik korrekt |
| `TILE_ALLOWED_WIDGET_TYPES` = Set(['gauge', 'sensor-card']) | `MonitorView.vue` | 934 | Existiert |
| `inlineMonitorPanelsForZone(zoneId)` | `dashboard.store.ts` | 891-896 | Filter korrekt |
| `compact` Prop auf InlineDashboardPanel | `InlineDashboardPanel.vue` | 33-35 | Funktioniert, max-height 120px |

**Das Problem ist rein datenseitig:** Es existiert kein Dashboard das ALLE Filter-Bedingungen gleichzeitig erfuellt:

```
inlineMonitorPanelsForZone(zoneId) filtert:
  1. target.view === 'monitor'
  2. target.placement === 'inline'
  3. scope === 'zone'
  4. zoneId === <gewuenschte-zone>

Zusaetzlich: getZoneMiniPanelId() prueft:
  5. panel.widgets.every(w => TILE_ALLOWED_WIDGET_TYPES.has(w.type))
     → ALLE Widgets im Dashboard muessen gauge ODER sensor-card sein
```

### Warum NICHT generateZoneDashboard() nutzen

Die bestehende `generateZoneDashboard()` Funktion (dashboard.store.ts Zeile 693) ist NICHT geeignet, weil:

1. **Falsche Widget-Typen:** Sie generiert ALLE Widget-Typen (line-chart, actuator-control, etc.). Aber `getZoneMiniPanelId()` verlangt dass ALLE Widgets im Dashboard vom Typ `gauge` oder `sensor-card` sind (`widgets.every()`). Ein einziges line-chart Widget wuerde den Filter brechen.

2. **Kollision mit bestehenden Dashboards:** Die Funktion prueft `layouts.value.find(l => l.autoGenerated && l.scope === 'zone' && l.zoneId === zoneId)` (Zeile 694). Es existieren bereits auto-generierte Zone-Dashboards (mit line-charts etc.). Ein neuer Call wuerde diese UEBERSCHREIBEN statt ein zweites zu erstellen.

3. **Signatur-Inkompatibilitaet:** Die Signatur ist `generateZoneDashboard(zoneId: string, devices: DeviceInfo[], zoneName: string)` — das zweite Argument ist ein `devices[]`-Array, NICHT der Zone-Name. Die Devices enthalten `esp_id`, `sensors[]`, `actuators[]`.

### Loesung: Eigene Mini-Dashboard-Erstellung

Da nur 1-2 Gauge-Widgets pro Zone noetig sind (nicht das volle Dashboard-Sortiment), erstellen wir Mini-Dashboards manuell per `createLayout()` + `addWidget()`. Zur Unterscheidung von normalen Zone-Dashboards nutzen wir `target.placement = 'tile'` (statt 'inline' der normalen Zone-Dashboards).

**ACHTUNG: Vor der Implementierung pruefen ob `inlineMonitorPanelsForZone()` auf `placement === 'inline'` filtert oder ob `getZoneMiniPanelId()` einen eigenen Filter hat.** Falls der Filter auf `'inline'` steht, muss er auf `'tile'` geaendert werden — ODER wir nutzen `'inline'` als placement und stellen sicher dass die Kollisionspruefung (Punkt 2 oben) funktioniert.

Konkret die Filter-Kette nochmal lesen:
- `_inlineMonitorBase()` (Zeile 881-882): Filtert auf `target.placement === ???`
- `inlineMonitorPanelsForZone()` (Zeile 891-896): Welches placement wird hier erwartet?
- `getZoneMiniPanelId()` (Zeile 940-948): Welches placement wird hier gesucht?

**Die placement-Wahl haengt davon ab was der bestehende Filter erwartet. Den Code lesen und entsprechend waehlen.**

### Technischer Fix

#### Schritt 1: Sensor-Daten pro Zone ermitteln

`filteredZoneKPIs` (aus `useZoneKPIs.ts`) hat `zoneId` und `zoneName`, aber KEINE Device- oder Sensor-Details. Die Sensoren muessen aus dem ESP-Store kommen.

In `MonitorView.vue`:

```typescript
import { useEspStore } from '@/shared/stores/esp.store'

function getSensorsForZone(zoneId: string): Array<{ espId: string, gpio: number, sensorType: string }> {
  const espStore = useEspStore()
  const sensors: Array<{ espId: string, gpio: number, sensorType: string }> = []

  for (const device of espStore.devices) {
    // Pruefe ob das Device dieser Zone zugeordnet ist
    if (device.zone_id !== zoneId) continue

    // Sammle alle Sensoren des Devices
    if (device.sensors) {
      for (const sensor of device.sensors) {
        sensors.push({
          espId: device.esp_id,
          gpio: sensor.gpio,
          sensorType: sensor.sensor_type
        })
      }
    }
  }

  return sensors
}
```

**ACHTUNG:** Die genaue Struktur von `espStore.devices` und den Sensor-Objekten muss verifiziert werden. Die Properties koennten anders heissen (z.B. `sensor_configs` statt `sensors`, `device_id` statt `esp_id`). Lies `esp.store.ts` und die Device/Sensor Interfaces bevor du den Code schreibst.

Auch pruefen: Gibt es bereits eine Hilfsfunktion die Devices nach Zone gruppiert? Z.B. `groupDevicesByZone()` in `useZoneDragDrop` — falls ja, diese wiederverwenden statt neu zu schreiben.

#### Schritt 2: Mini-Dashboard erstellen

```typescript
async function ensureZoneTileDashboard(zoneId: string, zoneName: string) {
  const dashStore = useDashboardStore()

  // Pruefen ob bereits ein Tile-Dashboard fuer diese Zone existiert
  const existingPanelId = getZoneMiniPanelId(zoneId)
  if (existingPanelId) return // Schon vorhanden

  // Sensoren der Zone holen
  const sensors = getSensorsForZone(zoneId)
  if (sensors.length === 0) return // Zone hat keine Sensoren → kein Dashboard

  // Temperatur- und Feuchtigkeits-Sensor finden (fuer die Standard-Gauges)
  const tempSensor = sensors.find(s =>
    s.sensorType.includes('temp') || s.sensorType.includes('temperature')
  )
  const humiditySensor = sensors.find(s =>
    s.sensorType.includes('humi') || s.sensorType.includes('humidity')
  )

  // Mindestens einen Sensor brauchen wir
  if (!tempSensor && !humiditySensor) return

  // Dashboard erstellen
  // ACHTUNG: createLayout() Signatur pruefen!
  // Aktuell: createLayout(name: string, description?: string): DashboardLayout
  // Falls createLayout() KEIN options-Objekt akzeptiert, muss das Layout
  // NACH dem Erstellen manuell mit scope/zoneId/target/autoGenerated angereichert werden.
  //
  // Alternative: Direkt ins layouts-Array pushen und syncToServer() aufrufen,
  // falls createLayout() zu eingeschraenkt ist.

  const layoutName = `${zoneName} Tile`

  // Option A: Falls createLayout() erweitert werden kann/soll:
  const layout = await dashStore.createLayout(layoutName, {
    scope: 'zone',
    zoneId: zoneId,
    target: { view: 'monitor', placement: 'tile' },  // oder 'inline' — siehe Pruefung oben
    autoGenerated: true
  })

  // Option B: Falls createLayout() NUR (name, description?) akzeptiert:
  // const layout = await dashStore.createLayout(layoutName)
  // layout.scope = 'zone'
  // layout.zoneId = zoneId
  // layout.target = { view: 'monitor', placement: 'tile' }
  // layout.autoGenerated = true
  // await dashStore.syncLayoutToServer(layout.id)  // Persists erweiterte Felder

  // Gauge-Widgets hinzufuegen
  if (tempSensor) {
    const sensorId = `${tempSensor.espId}:${tempSensor.gpio}:${tempSensor.sensorType}`
    await dashStore.addWidget(layout.id, {
      type: 'gauge',
      title: 'Temperatur',
      sensorId: sensorId,
      // Default-Config aus WIDGET_DEFAULT_CONFIGS['gauge'] nutzen fuer w/h
    })
  }

  if (humiditySensor) {
    const sensorId = `${humiditySensor.espId}:${humiditySensor.gpio}:${humiditySensor.sensorType}`
    await dashStore.addWidget(layout.id, {
      type: 'gauge',
      title: 'Luftfeuchtigkeit',
      sensorId: sensorId,
    })
  }
}
```

2. **`addWidget()` Signatur** in `dashboard.store.ts`: Akzeptiert sie `(layoutId, config)`? Was ist das genaue WidgetConfig-Interface? Welche Felder sind Pflicht (type, sensorId, title, w, h, x, y)?

#### Schritt 3: Auto-Generate beim L1-Laden aufrufen

In `MonitorView.vue`, nach dem bestehenden Datenlade-Code:

```typescript
// Guard: Nur einmal ausfuehren, nicht bei jedem Re-Render
let tileDashboardsEnsured = false

watch(filteredZoneKPIs, async (zones) => {
  if (tileDashboardsEnsured || zones.length === 0) return
  tileDashboardsEnsured = true

  for (const zone of zones) {
    await ensureZoneTileDashboard(zone.zoneId, zone.zoneName)
```
